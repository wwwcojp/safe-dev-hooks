# 実ユーザー名・実ホームパスの禁止

リポジトリ内のファイル(コード・テスト・ドキュメント・spec/plan・設定)に、実在するローカルユーザー名や実ホームパス(`/home/<実名>`、`/Users/<実名>`)を書かない。本リポジトリは公開を予定しており、混入すると履歴書き換え(filter-repo + force push)が必要になる。

## プレースホルダー規約

| 用途 | 使う表記 |
|---|---|
| ドキュメント・設定例のパス | `$HOME` または `/home/USER` |
| テストフィクスチャ | `/home/alice` |
| 一般的な説明 | `/home/<user>` |

## 強制レイヤー

この規約は `secrets_scan` の `custom_patterns`(`.claude-hooks.json` の `real-home-path`)とCIのリークチェックで機械的に検査される。ブロックされた場合は実パスをプレースホルダーに置き換えること。検査を無効化して回避しない。
